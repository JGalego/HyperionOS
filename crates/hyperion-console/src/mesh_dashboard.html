What am I looking at?

Each dot below is a separate, real Hyperion program running on this machine, each pretending to be a different device on the network. Every node only really knows how to do a few things (its capabilities are printed under its dot). When a node is asked to do something it can't do itself, it looks around the real network for a peer that can, checks that peer is really who it claims to be, and hands the task off to it -- no human names a host or port anywhere in this flow.

The real event log on the right is the plain-text version of the same story: every discovery, trust decision, and delegation, in the order it really happened.

  • ● a real, independently-running Hyperion node -- colored by which real backend it's dispatching through (see below).
  • ╌╌ this node has verified and now trusts that peer, from a real past delegation.
  • ➜ a dot really travels this edge while a discovery/delegation is in flight -- amber outbound (discovering), green returning (the real answer), red if it failed.
  • ◎ an expanding ring: this node was just asked to do something -- it honestly can't say by whom, since the protocol never authenticates its caller.
