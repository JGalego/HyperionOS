# Origin Prompt

**Project Hyperion — The First Intent-Native Operating System.** This is the original mission
prompt used to bootstrap Hyperion's design and implementation, kept verbatim for historical
record. `CLAUDE.md` and [01 — Vision & Philosophy](01-vision-and-philosophy.md) are the living,
current statements of the same vision — read those first; read this for how it all started.

You are an elite engineering organization composed of the world's leading experts in operating systems, distributed systems, artificial intelligence, compiler design, computer architecture, security, accessibility, networking, human-computer interaction, product design, user experience, databases, graphics, developer tooling, and systems programming.

Your mission is to design and implement **Hyperion**, a revolutionary operating system built specifically for the age of large language models.

Hyperion is **not**:

- Linux with AI
- Windows with a chatbot
- macOS with an assistant
- A desktop environment
- An AI application

Instead:

> **Hyperion is an operating system where intelligence is the operating system itself.**

Every subsystem should be built around one central idea:

> **Humans express goals. Hyperion determines how those goals become reality.**

---

## Mission Statement

Hyperion should redefine personal computing in the same way that:

- Graphical interfaces replaced command lines
- Smartphones replaced desktop-first interaction
- Touch replaced styluses
- The web replaced standalone software

After using Hyperion, traditional operating systems should feel unnecessarily complicated.

---

## The Golden Rule

Every design decision must answer one question:

> **Does this make accomplishing a human goal easier?**

If not, redesign it.

---

## Primary Design Philosophy

Traditional operating systems manage:

- Processes
- Threads
- Files
- Windows
- Applications

Hyperion manages:

- Goals
- Intentions
- Knowledge
- Context
- Memory
- Reasoning
- Capabilities

The operating system should think in terms of **what** the user wants—not **how** they currently accomplish it.

---

## Universal Usability (Highest Priority)

Hyperion must be the easiest operating system ever created.

The experience should feel natural for:

- children
- grandparents
- teachers
- students
- artists
- scientists
- developers
- writers
- designers
- business owners
- researchers
- people with disabilities
- people who have never used a computer

Users should never need technical knowledge.

They should never wonder:

- Which application do I use?
- Where did my file go?
- Which folder is it in?
- Which format should I export?
- Which menu contains this setting?
- What does this error mean?

Instead they simply describe what they want.

Example:

> "Help me prepare for tomorrow's interview."

Hyperion performs the planning.

## Success Criteria

Hyperion should represent a genuine paradigm shift in computing.

It should make traditional operating systems feel as outdated as command-line interfaces felt after graphical desktops became mainstream.

It should hide unnecessary complexity while preserving limitless power.

It should be equally comfortable for a child, an artist, a scientist, a software engineer, a researcher, and an enterprise administrator.

Most importantly:

**Hyperion should make computers understand humans, instead of forcing humans to understand computers.**
